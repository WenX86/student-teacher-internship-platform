# 生产部署与安全加固说明

## 1. 当前状态
当前项目已经具备开发联调能力，但进入生产前仍建议使用 `prod` profile，并按本文完成部署、安全与运维配置。

## 2. 生产启动建议
启动命令：
```powershell
java -jar .\target\internship-platform-backend-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

建议配置环境变量：
- `SERVER_PORT`
- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USERNAME`
- `DB_PASSWORD`
- `DB_SSL`
- `APP_SECURITY_ALLOWED_ORIGINS`

## 3. 安全加固项
- 生产环境关闭 `H2 Console`
- `CORS` 仅允许白名单域名访问，不再默认全开放
- 保持 `UTF-8` 强制编码，避免接口中文乱码
- 增加 `Content-Type-Options`、`Referrer-Policy`、`Permissions-Policy`、`Cache-Control` 等安全头
- 建议上线前把默认演示账号密码全部重置
- 建议把令牌有效期、密码策略、操作审计进一步配置化
- 文件上传能力正式上线前，需要补齐类型、大小、病毒扫描与重命名策略

## 4. 部署建议
- 前端与后端分离部署，前端走静态资源托管，后端独立进程或容器运行
- 数据库单独部署 MySQL，不使用 H2
- 反向代理层建议使用 Nginx，并配置 HTTPS
- 日志输出建议落盘并接入日志采集
- 定时备份数据库，至少保留日备份与周备份

## 5. 测试建议
上线前至少完成：
- 四角色登录与权限回归
- 指导申请、实习申请、表单提交流程回归
- 教师评价、学院确认、批量归档回归
- 风险预警、催办提醒、参数配置回归
- 接口异常、越权访问、失效登录态回归
