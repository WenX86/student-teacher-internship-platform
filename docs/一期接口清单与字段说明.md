# 一期接口清单与字段说明

## 基本说明

- 基础地址：`http://localhost:8080/api`
- 认证方式：除 `/auth/login`、`/health` 外，均需 `Authorization: Bearer <token>`
- 返回结构：

```json
{
  "success": true,
  "data": {}
}
```

## 认证与账户

### POST `/auth/login`

- 说明：账号密码登录
- 请求体：
  - `account`：登录账号
  - `password`：登录密码
- 返回字段：
  - `token`：访问令牌
  - `user.id`
  - `user.account`
  - `user.name`
  - `user.role`
  - `user.collegeId`
  - `user.mustChangePassword`
  - `user.lastLoginAt`
  - `user.profile`

### GET `/auth/me`

- 说明：获取当前登录用户信息

### POST `/auth/change-password`

- 说明：修改密码
- 请求体：
  - `newPassword`

### GET `/auth/login-records`

- 说明：获取当前用户登录日志

## 状态限制与联调约定（2026-03-28补充）

### 指导教师申请

- `POST /mentor-applications`
  - 学生可直接新建申请。
- `POST /mentor-applications/{id}/teacher-review`
  - 仅允许处理状态为 `待教师确认` 的申请。
  - 同一条申请教师确认/驳回后，不允许再次教师审核。
- `POST /mentor-applications/{id}/college-review`
  - 仅允许处理状态为 `待学院复核` 的申请。
  - 学院管理员只能复核本学院教师对应的申请。
  - 学院复核完成后，不允许重复复核。

### 实习申请

- `POST /internship-applications`
  - 学生必须先有 `已生效` 的指导关系，才能提交实习申请。
  - 被学院退回后，当前实现采用“新建一条新的实习申请”完成重提，而不是在原记录上直接改状态重提。
- `POST /internship-applications/{id}/review`
  - 仅允许处理状态为 `待学院审批` 的申请。
  - 学院管理员只能审批本学院学生的申请。
  - 已通过或已退回的申请不可重复审批。

### 表单提交、退回与归档

- `POST /forms`
  - `submit=false` 时创建为 `草稿`
  - `submit=true` 时创建为 `教师审核中`
- `PUT /forms/{id}`
  - 仅允许学生修改自己名下且状态为 `草稿 / 教师退回 / 学院退回` 的表单。
  - `教师审核中 / 学院审核中 / 已归档` 状态不可直接修改。
  - 退回后重提时，沿用原表单记录，版本号递增，历史写入 `history`。
- `POST /forms/{id}/teacher-review`
  - 仅允许处理状态为 `教师审核中` 的表单。
  - 审核通过后流转为 `学院审核中`
  - 审核退回后流转为 `教师退回`
- `POST /forms/{id}/college-review`
  - 仅允许处理状态为 `学院审核中` 的表单。
  - 学院管理员只能处理本学院学生的表单。
  - 审核通过后流转为 `已归档`
  - 审核退回后流转为 `学院退回`

### 学院入驻申请

- `POST /admin/college-applications/{id}/review`
  - 仅允许处理状态为 `待审核` 的入驻申请。
  - 审核通过后自动创建或补全学院，并在缺少账号时自动生成学院管理员账号。
  - 已通过或已驳回的申请不可重复审核。

### 消息通知约定

- 指导教师申请新建后：
  - 向教师发送 `待办提醒`，链接 `/teacher/mentor-requests`
- 指导教师教师审核通过后：
  - 向学院管理员发送 `待办提醒`，链接 `/college/mentor-relations`
- 指导教师教师/学院驳回后：
  - 向学生发送 `退回通知`
- 指导关系学院复核通过后：
  - 向学生发送 `审核结果`
- 实习申请提交后：
  - 向学院管理员发送 `待办提醒`，链接 `/college/internship-applications`
- 实习申请退回/通过后：
  - 向学生发送对应 `退回通知 / 审核结果`
- 表单提交或重提后：
  - 向指导教师发送 `待办提醒`，链接 `/teacher/reviews`
- 教师审核通过后：
  - 向学院管理员发送 `待办提醒`，链接 `/college/archive`
- 教师或学院退回表单后：
  - 向学生发送 `退回通知`，链接 `/student/tasks`
- 学院归档通过后：
  - 向学生发送 `审核结果`，链接 `/student/tasks`
- 教师提交评价后：
  - 向学院管理员发送 `待办提醒`，链接 `/college/evaluations`
- 学院确认评价后：
  - 向学生发送 `评价结果`，链接 `/student/results`
### 评价确认补充约定

- `POST /evaluations/{id}/college-confirm`
  - 仅允许确认 `submittedToCollege = true` 且 `confirmedByCollege = false` 的评价记录。
  - 已完成学院确认的评价不可重复确认。
- `POST /evaluations`
  - 当前实现下，若评价已经被学院确认，教师端不可再修改该评价。
- 前端联调建议：
  - 教师端对 `confirmedByCollege = true` 的评价仅展示“已确认”，不再显示编辑按钮。
  - 学院端对 `confirmedByCollege = true` 的评价仅展示“已确认”，不再显示确认按钮。
### 前端状态展示与按钮显隐约定（2026-03-28补充）

- 学生端 实习申请：仅当指导教师申请状态为 已生效 时显示可用的“新建申请”按钮；未生效时按钮禁用，并提示先完成指导教师申请。
- 学生端 核心表单：仅 草稿 / 教师退回 / 学院退回 状态允许继续编辑，其余状态统一显示只读标签。
- 教师端 评价管理：confirmedByCollege = true 的评价统一显示为 已确认，不再展示“编辑”按钮。
- 学院端 评价确认：仅 submittedToCollege = true && confirmedByCollege = false 的记录展示“确认结果”按钮，其他记录统一显示 已确认。
- 学院端 归档中心：仅 学院审核中 的表单允许勾选进入批量归档或批量退回。
- 消息中心：前端统一按 message.type 显示类型标签，按 ead 字段显示 未读 / 已读 状态标签。
- 状态标签颜色约定：
  - success：已通过、已生效、已归档、已确认、启用
  - warning：待审核、待教师确认、待学院复核、待学院审批、教师审核中、学院审核中、待确认、未读、待改密
  - danger：教师退回、学院退回、已驳回
  - info：草稿、未申请、待完善、已读、停用
