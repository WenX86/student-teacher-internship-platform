# 一期开发团队与代码规划

## 1. 一期目标

严格围绕需求文档 `13.1 一期` 落地以下闭环：

1. 登录认证与权限控制
2. 学生、教师、学院基础管理
3. 实习单位信息库管理
4. 指导关系建立与学院复核
5. 学生实习申请与学院审批
6. 任课实习、班主任实习核心表单
7. 教师审核与学院归档
8. 基础统计与消息提醒

## 2. 开发团队

### 2.1 产品与业务组

- 产品负责人：把控需求边界、流程闭环、验收口径
- 业务分析师：拆解指导关系、实习申请、表单审核三个核心流程
- 测试负责人：按角色和状态机设计验收场景

### 2.2 技术实现组

- 技术负责人：定义一期总体架构、状态机、数据模型和接口边界
- 后端负责人：认证、权限、档案、审批、审核、消息、统计
- 前端负责人：四角色工作台、业务页、路由、状态管理、联调
- 数据模型负责人：统一实体命名、状态枚举、版本留痕

### 2.3 智能体协同组

- 领域建模智能体：抽取一期实体、状态、接口分组
- 前端信息架构智能体：抽取菜单、路由、页面最小集
- 集成验证智能体：后续补充接口走查与回归清单

## 3. 一期领域拆分

### 3.1 核心实体

- 用户
- 学院
- 学生
- 指导教师
- 实习单位
- 指导申请
- 实习申请
- 表单模板
- 表单实例
- 消息通知
- 指导记录
- 评价记录
- 日志记录

### 3.2 关键状态机

指导申请：
- 待教师确认
- 教师驳回
- 待学院复核
- 学院驳回
- 已生效

实习申请：
- 待学院审批
- 已退回
- 已通过

表单实例：
- 草稿
- 教师审核中
- 教师退回
- 学院审核中
- 学院退回
- 已归档

## 4. 代码规划

### 4.1 本次工程实现

- `backend/`：一期业务中台接口，承载角色认证、业务流转、消息、统计
- `frontend/`：四角色门户与一期主流程页面
- `docs/`：团队分工、范围约束、后续迁移说明

### 4.2 先实现的接口分组

1. 认证与当前用户
2. 工作台与消息中心
3. 学生与教师基础管理
4. 实习单位信息库
5. 指导申请流转
6. 实习申请审批
7. 表单提交、教师审核、学院归档
8. 指导记录、评价、统计

### 4.3 前端实现策略

1. 先做登录和统一门户布局
2. 再做学生、教师、学院管理员三端主链页面
3. 最后补超级管理员基础页、报表页和消息页

## 5. 一期验收主链

1. 学院管理员维护教师、学生、实习单位
2. 学生申请指导教师
3. 教师确认，学院复核生效
4. 学生提交实习申请，学院审批
5. 学生提交任课或班主任核心表单
6. 教师审核通过或退回
7. 学院终审归档
8. 系统生成消息提醒和基础统计

## 6. 后续演进

- 当前实现优先保证文档逻辑闭环和接口边界清晰
- 后续可平滑迁移到文档建议的 Spring Boot + MyBatis-Plus + MySQL 组合
- 表单模板、自定义表单、批量归档、风险预警保留为二期扩展位
