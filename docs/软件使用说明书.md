# 软件使用说明书

## 1. 文档说明

本文档用于说明“师范生教育实习全过程管理平台”的软件用途、运行方式、角色功能、业务流程和试运行前准备事项。

- 软件名称：师范生教育实习全过程管理平台
- 文档性质：软件使用说明书
- 适用对象：学生、指导教师、学院管理员、超级管理员、项目实施人员
- 编写日期：2026年4月12日

## 2. 软件简介

本系统用于支撑师范生教育实习全过程线上管理，覆盖指导教师申请、实习申请、过程表单填报、教师审核、学院归档、指导记录、评价确认、消息提醒、统计报表等业务场景。

系统当前采用前后端分离架构。

- 前端：Vue 3 + Vite + Element Plus
- 后端：Spring Boot 3.3.5 + Spring Security + MyBatis-Plus
- 数据库：开发与测试阶段默认使用 H2，部署阶段建议使用 MySQL

## 3. 运行环境与访问方式

### 3.1 推荐环境

- 操作系统：Windows 10/11 或同类可运行 Java 21 与 Node.js 的环境
- JDK：21
- Maven：3.9.x
- Node.js：18 及以上
- 浏览器：Chrome、Edge

### 3.2 默认访问地址

- 前端开发访问地址：`http://localhost:5173`
- 后端接口地址：`http://localhost:8080/api`
- 后端健康检查：`http://localhost:8080/api/health`

### 3.3 启动方式

推荐使用仓库根目录脚本统一启动。

- 启动前后端：`.\start-all.ps1`
- 停止前后端：`.\stop-all.ps1`

如只启动后端，可进入 `backend-spring` 目录后执行：

- 前台启动：`.\start-dev.ps1`
- 后台启动：`.\start-dev.ps1 -Detached`
- 停止后端：`.\stop-dev.ps1`

### 3.4 生产部署相关配置

正式部署前需准备以下关键配置。

- `SERVER_PORT`
- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USERNAME`
- `DB_PASSWORD`
- `DB_SSL`
- `APP_SECURITY_ALLOWED_ORIGINS`
- 前端 `VITE_API_BASE`

## 5. 角色功能说明

### 5.1 学生端

学生端主要用于完成实习申请与过程材料提交。

- 首页看板：查看指导关系状态、实习状态、待办数量、归档数量
- 指导教师申请：发起指导教师申请，等待教师确认与学院复核
- 实习申请：在指导关系生效后提交实习申请，等待学院审批
- 表单中心：填写实习记录、实习心得、值守记录等过程表单
- 消息中心：查看退回通知、审核结果、催办提醒
- 结果查看：查看评价结果、归档结果、最终成绩

学生端使用顺序建议如下。

- 先提交指导教师申请
- 指导关系生效后再提交实习申请
- 实习申请通过后再提交过程表单
- 表单被退回后根据意见修改并重新提交

### 5.2 指导教师端

指导教师端主要用于审核学生材料与提交评价。

- 首页看板：查看待办审核数量和已归档统计
- 指导申请处理：审核学生提交的指导教师申请
- 材料审核：审核学生提交的过程表单
- 指导记录：登记指导时间、指导方式、问题与建议
- 评价管理：填写阶段评语、总结评语与成绩建议
- 预警提醒：查看超时未处理材料
- 消息中心：查看待办提醒与流程通知

### 5.3 学院管理员端

学院管理员端主要用于基础数据维护、流程终审与归档。

- 首页看板：查看学生数量、待审批数量、待归档数量、风险数量
- 学生管理：新增学生、重置密码、启停账号
- 教师管理：新增教师、重置密码、启停账号
- 指导关系复核：复核教师已确认的指导申请
- 实习单位管理：维护合作单位信息
- 实习申请审批：审批学生实习申请
- 归档中心：对教师审核通过的表单进行归档或退回
- 评价确认：确认教师提交的实习评价
- 预警中心：查看教师超时、学生超时、评价待确认等风险项
- 报表中心：查看统计分布与业务汇总
- 消息中心：查看提醒消息与处理结果

### 5.4 超级管理员端

超级管理员端主要用于平台基础配置。

- 首页看板：查看平台总体统计信息
- 学院与管理员管理：审批学院申请并配置学院管理员账号
- 参数配置：维护审核超时、预警级别、消息模板等系统参数
- 表单模板管理：新增、修改、启停表单模板
- 日志审计：查看登录日志和关键操作日志

## 6. 典型业务流程

### 6.1 指导关系建立流程

- 学生提交指导教师申请
- 指导教师审核
- 学院管理员复核
- 审核通过后指导关系正式生效

### 6.2 实习申请流程

- 学生在指导关系生效后提交实习申请
- 学院管理员审批实习申请
- 审批通过后学生进入“实习中”状态

### 6.3 表单提交流程

- 学生填写并提交过程表单
- 指导教师审核
- 教师通过后表单进入“学院审核中”
- 学院管理员归档或退回
- 已归档表单如需修改，学生需先提交修改申请

### 6.4 评价确认流程

- 指导教师提交实习评价
- 学院管理员确认最终成绩和评价意见
- 学生在结果页面查看最终评价

## 7. 状态说明

### 7.1 表单状态

- 草稿
- 教师审核中
- 教师退回
- 学院审核中
- 学院退回
- 已归档
- 修改申请中
- 允许修改

### 7.2 账号状态

- `ACTIVE`：可正常登录
- `DISABLED`：不可登录

### 7.3 实习申请状态

- 待学院审批
- 已通过
- 已退回

## 8. 常见问题

### 8.1 无法登录系统

- 检查账号和密码是否正确
- 检查账号是否被停用
- 如为重置密码后的首次登录，请使用默认密码后及时修改

### 8.2 无法提交实习申请

通常是因为指导关系尚未正式生效。

### 8.3 无法提交表单

系统要求学生先完成指导关系生效和实习申请审批通过后，才能提交或重新提交过程表单。

### 8.4 为什么教师不能审核某学生材料

教师只能审核与自己存在有效指导关系的学生材料。

## 9. 部署与试运行准备

### 9.1 试运行前准备

- 准备 JDK 21、Maven、Node.js、MySQL
- 确认前后端端口未被占用
- 准备正式域名或试运行访问地址
- 确认数据库字符集使用 UTF-8
- 确认浏览器可访问前端与后端接口

### 9.2 部署建议

- 前端使用 `npm run build` 生成静态资源后部署到 Nginx 或同类静态服务
- 后端使用 `mvn package` 打包后以 `prod` 配置启动
- 生产环境关闭 H2 Console，改用 MySQL
- 使用 HTTPS 和反向代理
- 配置日志落盘、数据库备份、默认密码重置

### 9.3 试运行建议

建议至少完成以下试运行检查。

- 四类角色均可正常登录
- 指导申请流程可完整流转
- 实习申请审批可完整流转
- 表单提交、教师审核、学院归档可完整流转
- 评价提交流程可完整流转
- 消息提醒和结果通知可正常查看
- 报表统计页面可正常加载

## 10. 使用建议

- 学生应按“指导关系 -> 实习申请 -> 过程表单 -> 评价结果”的顺序使用系统
- 教师与学院管理员应及时处理待办，减少超时预警
- 管理员应在试运行前统一重置演示账号默认密码
- 正式上线前建议根据学院实际业务继续补充真实数据、真实组织信息与表单模板
