## 第2页 项目背景

### 页面标题

项目背景与问题分析

### 页面展示要点

- 传统教育实习管理依赖纸质材料和人工登记
- 流程分散，沟通成本高
- 材料提交、审核、反馈缺乏统一平台
- 数据沉淀不足，统计分析困难

### 参考讲稿

师范生教育实习是人才培养中的重要实践环节，但传统管理方式通常依赖纸质材料、Excel 台账和线下沟通，存在流程分散、审核低效、数据难追踪等问题。尤其在学生材料提交、教师指导审核、学院归档和成绩评价等环节，常常因为信息不统一而影响管理效率。因此，建设一个覆盖全过程的数字化管理平台具有较强的现实意义。

项目建设目标

## 第4页 系统角色与功能

### 页面标题

系统角色与核心功能

### 页面展示要点

- 学生：申请、填报、查看结果
- 指导教师：审核材料、记录指导、提交评价
- 学院管理员：审批申请、归档材料、确认评价
- 超级管理员：基础配置、模板管理、日志审计

### 参考讲稿

系统围绕四类核心角色展开设计。学生负责申请和表单填报；指导教师负责审核和指导记录；学院管理员负责审批、归档和评价确认；超级管理员负责系统级配置与管理。通过这样的角色划分，系统能够保证权限清晰、职责明确，也使各个业务环节衔接更加顺畅。

## 第5页 业务流程

### 页面标题

核心业务流程

### 页面展示要点

- 指导教师申请
- 实习申请审批
- 过程表单提交与审核
- 学院归档与评价确认

### 页面可视化建议

建议使用流程图展示：

- 学生提交指导申请
- 教师确认
- 学院复核
- 学生提交实习申请
- 学院审批
- 学生提交表单
- 教师审核
- 学院归档
- 教师提交评价
- 学院确认评价

### 参考讲稿

平台的业务流程可以概括为四个阶段。第一，学生提交指导教师申请，由教师确认、学院复核后生效。第二，学生在指导关系生效后提交实习申请，由学院管理员审批。第三，学生进入实习阶段后，按要求提交过程表单，教师审核通过后进入学院归档。第四，教师提交评价，学院确认最终结果。这样就形成了完整的业务闭环。

## 第6页 技术方案

### 页面标题

系统技术架构

### 页面展示要点

- 前后端分离架构
- 前端：Vue 3 + Vite + Element Plus
- 后端：Spring Boot + Spring Security + MyBatis-Plus
- 数据库：H2 / MySQL
- 支持本地联调与后续部署扩展

### 参考讲稿

在技术实现上，本项目采用前后端分离架构。前端使用 Vue 3、Vite 和 Element Plus 构建管理界面，后端采用 Spring Boot、Spring Security 和 MyBatis-Plus 实现认证授权和业务逻辑。开发测试阶段使用 H2 数据库，部署阶段可切换到 MySQL。这样的架构既保证了开发效率，也为后续扩展和维护提供了良好基础。
